JASPAR IN BRIEF

JASPAR is a collection of transcription factor DNA-binding preferences, modelled as position-specific weight matrices (PSSMs).

The prime difference to similar resources (TRANSFAC, TESS etc) consist of non-redundancy and quality: JASPAR is a smaller set that is non-redundant and curated

All profiles are derived from published collections of experimentally defined transcription factor binding sites for multicellular eukaryotes. The database represents a curated collection of target sequences. The binding sites were determined either in SELEX experiments, or by the collection of data from the experimentally determined binding regions of actual regulatory regions; this distinction is clearly marked in the profiles' annotation. As far as possible, the collection is non-redundant (several models describing one transcription factor)


PROFILE COLLECTION METHODOLOGY

Profiles were collected using the following steps:

1 Location of published work describing either a SELEX experiment or a collection of in-vivo binding sequences for a transcription factor.

2 Assessment of paper: Continue if

The methodology is acceptable, the work is of high quality

AND no filtering was used except possibly checks for site activity in vivo

AND sites are available (with some exceptions)

AND the transcription factor is from a multicellular organism

AND the transcription factor is not already in the database

3 Alignment of sites to produce profile

Bound sites for the TF were extracted using OCR software. We used the ANN_SPEC motif finding program (1) to align the binding sites. As it uses a probabilistic algorithm, the results may vary slightly; also, the width of the pattern is a variable. Thus three seeds were used, 80.000 iterations performed using several different widths. A relevant width was then selected using biological knowledge and visual inspection.

The start page is a gateway for profile retrieval. Different search methods are accessible from here, as well as links to download and information pages

SEARCH MODES

Often a subset of profiles are of interest for the working researcher. We provide several different ways of retrieval:

Browse

For browsing the entire collection of profiles, push any button in the browse field to view all profiles sorted by that attribute.

Search by...

For selection of subsets of profiles using user set criteria, use the search by fields.

Multiple inputs are acceptable, if submitted with commas in between ',' (this will in effect be interpreted as an OR statement).

The criteria will be interpreted from top to bottom by using the boolean statement at each row

an AND statement will perform an intersect between two query results

an OR statement will perform a union of two query results

an NOT statement will filterout results from the first query

For example, a search using the following criteria

will give all profiles that are of the selex type from human but not belonging to the bHLH or bZIP classes

UPLOAD set of profiles

If saved at an earlier session, a file with a list of profile IDs can be submitted for viewing

COMPARE custom profile to database profiles

In some cases, it is beneficial to assess similarity to input data (as with using BLAST for sequence data comparison when using Genbank).

A user submitted profile can be submitted in the format

12 14  2  3  0
 2  0 10  2  0
 0  0  1  5  0
 0  0  1  4 14

where rows corresponds to nucleotides and columns to positions.So, the above matrix is refering to 

   1  2  3  4  5
A 12 14  2  3  0
C  2  0 10  2  0
G  0  0  1  5  0
T  0  0  1  4 14

The input profile can consist of actual counts or be normalized (each column sum =1), but not in the form of a PWM.

All profiles in the database will be compared to the input profile, using a modified Needleman-Wunsch algorithm described in (2), and sorted by raw comparison score (for reference, the maximum score is 2*the width of the smallest matrix in the compared pair). Both the score and fraction of potential maximal score is reported. More detailed descriptions of the comparison algorithm can be found in refrences 2 and 3.



LIST PAGE

The list page presents a list of selected profiles.

For each retrieved profile, basic attributes are displayed (ID, Name, species, class) together with a sequence logo for visual inspection. For detailed information regarding any profile, press the view link.

Export profiles

Subsets of profiles can be exported, either as a simple list of IDs for simplified retrieval in subsequent database session or for usage in the CONSITE system for phylogenetic footprinting, or as a compressed flatfile representation (readable with the TFBS::DB::FlatFileDir module)

Analyze sequence

Using a subset of profiles, a submitted sequence can be analyzed. Sensitivity and selectivity will be affected by the relative score threshold, by default 80%. The analysis is simple: for high selectivity the consite system should be used (see above)



Detailed information page

This smaller window displays a sequence logo, the actual count profile and detailed information about the selected profile, where the fields corresponds to

Logos are a visual representation representation of a profile, based on Shannon information content. For DNA profiles, maximal conservation amounts to a information content of 2 bits for a single position.

Profiles are represented as a simple matrix where rows corresponds to nucleotide type and columns to positions

ID The ID used in the database

NAME Name of the transcription factor (this is somewhat arbitrary, as many factors have several names)

SPECIES The species of the transcription factor. In some cases, multiple species are used. It should also be noted that most binding interfaces are heavily conserved throughout evolution; thus usage of a chicken model when analyzing a human sequence is usually acceptable.

INFORMATION CONTENT The shannon information constent of the profile, measured in bits. Higher conservation at positions gives higher specificity and high information content.

SUPERGROUP simplified taxonomic groups, vertebrates, plants and insects

PROTEIN SEQ accession number and link to relevant protein sequence of the TF

EXPERIMENT Either a SELEX type or a compilation of in-vivo sites

SOURCE PMID PMID number and link to Pub med database for accessing the source publication

SAVE MODEL Save the model as a compressed flat file (readable with the TFBS::DB::FlatFileDir module)

REVERSE COMPLEMENT for convenience, profiles can be reverse complemented (the orientation of a matrix is arbitrary)



Sequence analysis page

This page presents the putative binding sites predicted using he selected profile on the submitted sequence, given the relative score threshold.The sequence analysis avaliable in the JASPAR interface is intended for spurios searches, and has few features. We recommend using the ConSite interface for advanced searches (including cross-species comparison). ConSite uses the JASPAR database for profile selection.

The same information will be displayed as

- a simple image where flags corresponds to putative binding sites,

- a more detailed table

- a sequence view



Graphics and tables are eqivalent to ConSite analysis output in single sequence mode. Sites are linked to a pop-up windows showing profile logo, the site location in the sequence and score.
